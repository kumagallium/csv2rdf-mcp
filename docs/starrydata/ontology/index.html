Starrydata Ontology

Phase 1 Apache-2.0 RDFS / OWL TBox

This namespace defines classes and properties used by the csv2rdf-mcp project to represent the Starrydata dataset (digitised measurement curves from thermoelectric, battery, and magnetic materials publications) as a provenance-rich knowledge graph.

Classes

IRI fragmentLabelDescription
#Paper ¶PaperA scholarly article that contains digitised measurement data. Subclass of schema:ScholarlyArticle and prov:Entity.
#Sample ¶SampleA physical specimen prepared and measured in a paper. Subclass of prov:Entity.
#Curve ¶CurveA measurement curve digitised from a figure in the source paper. Subclass of prov:Entity. Phase 1 stores raw x/y arrays as JSON literals plus pre-computed aggregates (xMin/xMax/yMin/yMax/pointCount).
#Descriptor ¶DescriptorStructured metadata extracted from the sample_info column (MaterialFamily, Form, FabricationProcess, etc.).
#IngestionActivity ¶Ingestion ActivityA single run of the csv2rdf-mcp ingester. Subclass of prov:Activity. Each Paper / Sample / Curve generated in the run carries prov:wasGeneratedBy → IngestionActivity.
#DigitizationActivity ¶Digitization ActivityThe act of extracting a curve's data points from a published figure (e.g. with WebPlotDigitizer). Subclass of prov:Activity. Records the scientific origin of the numbers (distinct from the pipeline run); a Curve is prov:wasGeneratedBy both. prov:atTime carries the curator timestamp when parseable.

Complete definitions — with rdfs:label / rdfs:comment in en and ja — are in ontology.ttl. The formal shape grammar (ShEx) is shipped alongside the data in data/togomcp/mie/starrydata.yaml.

IRI design

The prefix sd: resolves to this page; instance IRIs use the sdr: prefix (../resource/). Phase 1 mints instance IRIs as composite keys so that the raw sample_id / figure_id columns of Starrydata — which are not globally unique (collide across papers) — never collapse separate measurements into a single resource:

sd:   https://kumagallium.github.io/csv2rdf-mcp/starrydata/ontology#
sdr:  https://kumagallium.github.io/csv2rdf-mcp/starrydata/resource/

sdr:paper/{SID}
sdr:sample/{SID}-{sample_id}
sdr:curve/{SID}-{figure_id}-{sample_id}
sdr:descriptor/{SID}-{sample_id}/{idx}
sdr:periodical/{slug-of-journal-name}
sdr:person/{SID}/{author-idx}
sdr:ingestion/run-{timestamp}

Phase 2 will redirect https://w3id.org/csv2rdf/... here so the IRI strings can survive future host migrations.
