Starrydata Resource IRIs

This is the IRI prefix space for instances of Starrydata entities (papers, samples, curves, descriptors, periodicals, persons, ingestion activities). The full IRI of a single resource has the form:

https://kumagallium.github.io/csv2rdf-mcp/starrydata/resource/{type}/{id}

# Examples
https://kumagallium.github.io/csv2rdf-mcp/starrydata/resource/paper/61
https://kumagallium.github.io/csv2rdf-mcp/starrydata/resource/sample/61-25267
https://kumagallium.github.io/csv2rdf-mcp/starrydata/resource/curve/61-20485-25275

Dereferencing status

Phase 1: instance IRIs are not dereferenceable (they currently return this page; a direct visit to e.g. /resource/paper/61 shows 404). The IRIs are identifiers that work correctly in SPARQL / RDF semantics; they are not yet HTTP-resolvable.

Phase 2 plans to add a small dynamic service that responds to instance IRI requests with a SPARQL DESCRIBE result (Turtle for machines, HTML for humans, content-negotiated).

Where to query the data today

  • Public hosted SPARQL endpoint: (none yet — Phase 1 deployments are self-hosted on closed servers per the sovereignty-first design)
  • Run locally: docker compose up -d --build on the csv2rdf-mcp repository, then POST a Turtle file to http://localhost:7878/store?default
  • Query via MCP: register the togomcp container in any MCP-aware AI client (Claude Code, Cursor, Dify) — see the Crucible registration recipe

For the ontology definitions, see ../ontology/.
